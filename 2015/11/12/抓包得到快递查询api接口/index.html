抓包得到快递查询api接口

发表于 2015-11-12   |   分类于 代码琐事   |  

最近打算自己实现一个查询快递的简单App，于是上网查了有没有方便快捷的api接口。结果绝大部分都不尽人意。

不是申请流程麻烦，就是套餐价格死贵。

没办法，只能自力更生了。

最简便的方法就是找一家提供快递查询服务的网站，在Chrome浏览器上打开开发者工具，通过抓包查看网站本身是向什么地址请求的数据。

我选择了快递100这家网站。通过抓包发现，其主要通过两个接口实现快递查询。

  • 第一个是自动检测快递单号是属于哪个快递公司的接口：

Request URL:http://www.kuaidi100.com/autonumber/autoComNum?text=xxxx

参数 text：快递单号

  • 第二个便是查询主要快递信息的地址了：

Request URL:http://www.kuaidi100.com/query?type=xxxx&postid=xxxx

参数 type: 快递公司代号(这个可以从第一个接口返回的数据得到) postid: 快递单号

###以下是示例

Request: http://www.kuaidi100.com/autonumber/autoComNum?text=3100735084693

Return:

Request: http://www.kuaidi100.com/query?type=yunda&postid=3100735084693&id=1&valicode=&temp=0.06704130209982395

Return:

Over.
